## ?? Описание проекта
**TaskManager** — система электронного документооборота, предназначенная для автоматизации обработки документов: от создания/получения до отправки корреспонденту или завершения исполнения. Система оптимизирована для управления документами в рамках отдела организации.

---

## ?? Основные функции
- **Регистрация документов:**
  - Входящие, исходящие, внутренние документы.
- **Учет и контроль:**
  - Учет резолюций руководства.
  - Постановка документов на контроль.
  - Мониторинг сроков выполнения.
- **Архивация и отчетность:**
  - Списание документов в дело.
  - Формирование делопроизводственных отчетов.

---

## ?? Функциональные требования
1. **Базы данных:**
   - Основная БД для документов.
   - БД исполнителей/заказчиков (наименование, местоположение, контактные данные).
2. **Главный экран:**
   - Отображение актуальных задач (статус ? "завершено").
   - Сортировка по времени выполнения и приоритету.
3. **Авторизация:**
   - Администратор: полный доступ.
   - Пользователи: единый вход через логин ОС Windows.

## ?? Интерфейс и визуализация
- **Сортировка и фильтрация:**
  - По исходящему/входящему номеру, сроку выполнения, исполнителю.
- **Цветовая индикация сроков:**
  - ?? Зеленый: 7+ дней.
  - ?? Желтый: 4-6 дней.
  - ?? Красный: 0-3 дня.
- **Визуальные особенности:**
  - Задачи на контроле: **жирный шрифт**, отображаются в первых рядах.
  - Архивные задачи: скрыты (доступны через поиск).

## ?? Система ролей и доступов
| Роль               | Права                                                                 |
|--------------------|-----------------------------------------------------------------------|
| **Обычный пользователь** | Создание, просмотр, редактирование и удаление **своих** записей.     |
| **Администратор**        | Полный доступ, управление удаленными записями, восстановление, статистика. |

---

## ?? Работа с записями
- **Типы полей:**
  - Изменяемые владельцем.
  - Изменяемые админом.
  - Изменяемые другими работниками (информация о закрытии задачи).
- **Статусы:**
  - **Актуальная** (редактирование, главный экран).
  - **Выполнено** (только чтение, архив).
  - **Удаленная** (доступно админу).

---

## ??? Процесс работы
1. **Запуск программы:** Отображение актуальных задач с сортировкой по приоритету.
2. **Просмотр задачи:** Детализация записи с учетом прав доступа.
3. **Создание/редактирование:** Автоматическое владение при создании.
4. **Завершение задачи:** Статус "Выполнено" ? архив.
5. **Поиск и экспорт:** Поиск по всем записям, экспорт в `.txt`.

---

## ?? Технические требования
- **Авторизация:** Windows Authentication для пользователей, логин/пароль для админа.
- **Безопасность:** Журналирование действий, резервное копирование.
- **Производительность:** Быстрый поиск, оптимизированный интерфейс.

## ??? Структура базы данных
### Основные таблицы:
- **Tasks** (Записи): `id`, `title`, `owner_id`, `status`, `priority`, и др.
- **Users** (Пользователи): `id`, `username`, `role`, `department`.
- **Clients** (Исполнители/Заказчики): `id`, `name`, `contact_person`, `email`.
- **TaskHistory** (История изменений): `id`, `task_id`, `field_name`, `old_value`, `new_value`.

## ??? Установка и запуск
### Предварительные требования:
- .NET Framework 4.7.2+
- SQL Server 2016+
- Windows 7/8/10/11
- IIS (для веб-версии)
- Доступ к Active Directory

### Шаги установки:
1. **Установка БД:** Восстановить backup, настроить connection string.
2. **Установка приложения:** Скопировать файлы, настроить конфигурацию.
3. **Настройка авторизации:** Windows Authentication, создать учетную запись админа.
4. **Тестирование:** Проверка подключения к БД, создание записей, поиск и экспорт.
